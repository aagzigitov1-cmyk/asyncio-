День 6 — Сохранение данных и работа с файлами

🎯 Цель дня
Реализовать асинхронное сохранение собранных данных в файлы и базы данных.

📋 Требования

1. Установка библиотек
```bash
pip install aiofiles aiosqlite  # или asyncpg для PostgreSQL
```

2. Класс DataStorage
Создать абстрактный базовый класс:
- async def save(self, data: dict) — сохранение данных
- async def close(self) — закрытие соединения
- Поддержка различных форматов (JSON, CSV, SQLite)

3. JSONStorage
Реализовать сохранение в JSON:
- async def save(self, data: dict) — добавление в JSON-файл
- Использование aiofiles для асинхронной записи
- Поддержка форматированного вывода
- Обработка больших объёмов данных (построчная запись)

4. CSVStorage
Реализовать сохранение в CSV:
- Автоматическое определение заголовков
- Асинхронная запись строк
- Поддержка различных кодировок
- Обработка специальных символов

5. SQLiteStorage (или PostgreSQL)
Реализовать сохранение в БД:
- async def init_db(self) — создание таблиц
- async def save(self, data: dict) — вставка данных
- Использование aiosqlite или asyncpg
- Поддержка batch-вставок для производительности
- Индексы для быстрого поиска

6. Структура данных
Стандартизировать формат сохраняемых данных:
```python
{
    "url": str,
    "title": str,
    "text": str,
    "links": list[str],
    "metadata": dict,
    "crawled_at": datetime,
    "status_code": int,
    "content_type": str
}
```

7. Интеграция с AsyncCrawler
Добавить в AsyncCrawler:
- storage: DataStorage — объект для сохранения
- Автоматическое сохранение после обработки каждой страницы
- Batch-сохранение для оптимизации (опционально)

8. Обработка ошибок сохранения
- Повторы при ошибках записи
- Логирование ошибок сохранения
- Продолжение работы при ошибках сохранения (не падать)

9. Производительность
Оптимизировать сохранение:
- Batch-вставки в БД
- Буферизация для файлов
- Асинхронная запись без блокировки основного потока

10. Демонстрация
Создать скрипт, который:
- Краулит сайт
- Сохраняет данные в JSON, CSV и SQLite
- Показывает статистику по сохранённым данным
- Демонстрирует чтение сохранённых данных

Пример использования:
```python
# JSON storage
json_storage = JSONStorage("results.json")
crawler = AsyncCrawler(storage=json_storage)

# SQLite storage
db_storage = SQLiteStorage("crawler.db")
crawler = AsyncCrawler(storage=db_storage)

await crawler.crawl(start_urls=["https://example.com"])
await crawler.close()
```

11. Тестирование
- Протестировать сохранение в JSON
- Протестировать сохранение в CSV
- Протестировать сохранение в БД
- Протестировать обработку ошибок записи
- Проверить целостность сохранённых данных

✅ Критерии успеха
- Данные сохраняются асинхронно
- Поддерживаются несколько форматов
- Ошибки сохранения обрабатываются
- Производительность сохранения оптимизирована
- Сохранённые данные можно прочитать и использовать
