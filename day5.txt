День 5 — Обработка ошибок и автоматические повторы

🎯 Цель дня
Реализовать надёжную систему обработки ошибок с автоматическими повторами и экспоненциальным backoff.

📋 Требования

1. Класс RetryStrategy
Создать класс для управления повторами:
- __init__(self, max_retries: int = 3, backoff_factor: float = 2.0, retry_on: list = None)
- async def execute_with_retry(self, coro, *args, **kwargs) — выполнение с повторами
- 📈 Экспоненциальный backoff между попытками
- ⚙️ Настраиваемые типы ошибок для повтора

2. Классификация ошибок
Определить типы ошибок:
- ⏰ TransientError — временные ошибки (таймауты, 503, 429)
- ❌ PermanentError — постоянные ошибки (404, 403, 401)
- 🌐 NetworkError — сетевые ошибки (connection refused, DNS)
- 📄 ParseError — ошибки парсинга

3. Стратегии повторов
Реализовать разные стратегии:
- 🔄 Повторять только TransientError и NetworkError
- 🚫 Не повторять PermanentError
- 🔢 Ограниченное количество повторов для каждого типа
- ⚡️ Разные backoff для разных типов ошибок

4. Интеграция с AsyncCrawler
Добавить в AsyncCrawler:
- retry_strategy: RetryStrategy
- 🔄 Автоматические повторы при ошибках загрузки
- 📝 Логирование всех попыток
- 💾 Сохранение информации об ошибках

5. Обработка HTTP статусов
- 🚫 429 (Too Many Requests) — повторить с увеличенной задержкой
- 🔄 503 (Service Unavailable) — повторить
- ❌ 404 (Not Found) — не повторять, пометить как постоянную ошибку
- 🚫 403 (Forbidden) — не повторять
- ⚠️ 500 (Server Error) — повторить ограниченное количество раз

6. Таймауты и их обработка
- ⏱️ Настраиваемые таймауты (connect, read, total)
- 📈 Увеличение таймаутов при повторах
- ⚠️ Обработка asyncio.TimeoutError

7. Circuit Breaker (опционально)
Реализовать простой circuit breaker:
- 📊 Отслеживание частоты ошибок для домена
- 🚫 Временная блокировка домена при высокой частоте ошибок
- ✅ Автоматическое восстановление через некоторое время

8. Детальное логирование ошибок
Логировать:
- 🏷️ Тип ошибки
- 🔗 URL, на котором произошла ошибка
- 🔢 Номер попытки
- ⏰ Время до следующей попытки
- 🎯 Итоговый результат (успех/неудача)

9. Статистика ошибок
Вести статистику:
- 📊 Общее количество ошибок по типам
- ✅ Количество успешных повторов
- ⏱️ Среднее время на повтор
- 📋 Список URL с постоянными ошибками

10. Демонстрация
Создать скрипт, который:
- 🌐 Краулит сайты с различными типами ошибок
- 🔄 Демонстрирует автоматические повторы
- 📊 Показывает статистику по ошибкам
- 📄 Сохраняет отчёт об ошибках

Пример обработки:
```python
retry_strategy = RetryStrategy(
    max_retries=3,
    backoff_factor=2.0,
    retry_on=[TransientError, NetworkError]
)

# Автоматически повторит при таймауте или 503
result = await retry_strategy.execute_with_retry(
    crawler.fetch_url, "https://example.com"
)
```

11. Тестирование
- ⏰ Протестировать повторы при таймаутах
- 🔄 Протестировать повторы при 503
- 🚫 Протестировать отсутствие повторов при 404
- 📈 Протестировать экспоненциальный backoff
- 📊 Проверить статистику ошибок

🎉 Критерии успеха
- ✅ Ошибки классифицируются правильно
- 🔄 Повторы работают для временных ошибок
- 🚫 Постоянные ошибки не повторяются
- 📈 Экспоненциальный backoff применяется
- 📊 Статистика ошибок ведётся корректно
